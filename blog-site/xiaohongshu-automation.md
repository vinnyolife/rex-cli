## 核心技术：RexCLI 是怎么做到的？

你可能问了：这不就是一个脚本吗？跟 RexCLI 有啥关系？

好问题。我来告诉你区别。

### 浏览器自动化 + 上下文记忆

RexCLI 解决了两个核心问题：

**问题一：浏览器操作**
小红书是前端渲染的动态页面，普通脚本根本搞不定。RexCLI 内置了 Playwright MCP，直接控制真实浏览器，跟你手动操作没区别。

**问题二：上下文丢失**
以前跑脚本，半途而废怎么办？重启后从哪开始？RexCLI 有 Context DB 会话记忆功能，脚本中断后可以从断点继续，不会重复操作。

### 生活类比

这么说吧：普通脚本就像一次性纸巾，用完就丢。RexCLI 就像智能管家，不仅帮你干活，还**记得住你上次干到哪了**。

## 怎么实现的？（极简版）

```bash
# 1. 克隆 RexCLI
git clone https://github.com/rexleimo/rex-cli.git

# 2. 一键安装
cd rex-cli
./scripts/setup-all.sh --components all

# 3. 启动浏览器自动化
codex
```

然后写一个简单的任务配置：

```json
{
  "task": "xiaohongshu-auto-post",
  "actions": [
    {"type": "login"},
    {"type": "publish", "images": "./images/", "content": "你的文案"},
    {"type": "interact", "target": "followers", "action": "like-comment"}
  ],
  "schedule": "0 9 * * *"
}
```

具体脚本和配置可以在 [rexai.top](https://rexai.top) 获取，这里不展开。

## 常见坑与对策

### 坑1：账号被风控

小红书对自动化操作有风控体系，别傻乎乎地 1 秒操作 10 次。

**对策**：在 RexCLI 配置里加入随机延迟，每次操作间隔 5-30 秒随机模拟人类行为。

### 坑2：登录态丢失

跑久了 Cookie 过期，浏览器登录态没了。

**对策**：RexCLI 支持保存浏览器配置文件，定期检查登录态，快过期时提前重新登录。

### 坑3：内容被判重复

同一个文案发多个号，被识别为重复内容。

**对策**：配置文案变体生成，同一个意思换三种说法，轮流使用。
