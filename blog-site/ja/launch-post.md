# 遂に登場！AIプログラミングツールの「戦い」を解決するRexCLIを作りました

正直に言うと、かつて3つのAIプログラミングツールに苦しんでいた。

Claude Codeはコードが強いが、ブラウザ自動化の設定が面倒。
Codex自動化は快適だが、複雑なコードリファクタリングは一息。
Gemini資料検索は良いが、仕事させるとよく落ちる。

「なぜ一つのツールで全てを網羅できないのか？」ずっと不思議だった。

 그래서自分で作った。

---

## 01. どんな問題に遭遇した？

### シーン1：ブラウザ自動化

小红書の自動投稿想让したい。

Claude Codeを使う？まずMCPを設定。3時間かけて漸く動くようになった。なのに次の日—アカウントが風控された。

Codexを使う？箱から出してすぐ使えてлання. 但途中でClaudeにコードを最適化させたかったら—**コンテキスト全部消失**、最初からやり直し。

### シーン2：長タスク中断

2000行以上のコードリファクタリング任务を実行中。会議に出て戻る—

Claude：どこだっけ、覚えてるが细节忘れた。
Codex：どこだっけ、リファクタリング思路ずれた。
Gemini：私は誰、何処？

**これが当時の日常：ツールを切り替えるか、設定を再構成するか。**

---

## 02. RexCLIとは？

**RexCLI = Claude Code + Codex + Gemini を協働させる**

### コア機能

| 機能 | 意味 |
|------|------|
| 統一ブラウザ自動化 | どのCLIでも同じ`browser_*`コマンド |
| 跨CLIコンテキスト記憶 | ツール切替でも進捗 保存 |
| Privacy Guard | 設定ファイル自動脱敏、API Key漏洩防止 |

### 仕組み

```
codex/claude/geminiを入力
       ↓
RexCLIが自動キャプチャ
       ↓
判断：ラップするか透伝するか？
       ↓
ラップ：ContextDB + Browser MCP接入
透伝：原生ツールに直接渡す
```

**操作習慣を変える必要なし。** 今まで通り使えば良い。

---

## 03. 実際の効果

### 効果1：ブラウザ自動化

```bash
# 昔：Codexを使う
codex

# 今：Codexsを使う but能力が違う
codex
```

的区别在于 今は統一された`browser_*`工具が使える：
- `browser_navigate` - ページを開く
- `browser_click` - 要素をクリック
- `browser_snapshot` - ページ内容を取得
- `browser_screenshot` - スクリーンショット

**どのCLIに変わっても、これらのコマンドが使える。**

### 効果2：ブレークポイント续跑

タスクの半分頑張って、ツールを切り替え？

```bash
# Codexで半分実行
codex

# Claudeに切り替えて続ける、コンテキスト自動同期
claude
```

**コピー＆ペースト不要、タスクを再説明不要。**

## 05. なぜオープンソース？

知られていると思う：こんなにツールがあるのに、なぜまだ作るの？

**因为他，比我 更懂这种"工具分裂"的痛。**

与其忍耐，不如自己动手。

**RexCLIは私の個人作品，也是我每天都在用的工具。**
