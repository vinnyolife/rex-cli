# ついに登場！AIプログラミングツールの「戦い」を解決するRexCLIを作りました

正直に言うと、かつて3つのAIプログラミングツールにさんざんな目に遭っていました。

Claude Codeはコーディング能力强だが、ブラウザ自動化の設定が面倒。
Codex自動化は心地よいが、複雑なコードリファクタリングはもうつ及ばない。
Gemini资料検索は良いが、仕事させると 자주 문제가 발생한다.

「なぜつのツールで全てをカバーできないのか？」ずっと思議でした。

 그래서、自分で 만들었다.

---

## 01. どんな問題に遭遇したか？

### シーン1：ブラウザ自動化

小红書の自動投稿したかった.

Claude Codeを使う？まずMCPの設定。3時間かけてようやく動いた。翌朝きて見ると——アカウントが风控された.

Codexを使う？箱から出してすぐ使えて素晴らしい。だが途中でClaudeにコード优化시켜주고 싶었으면——**コンテキスト全部了**、最初から。

### シーン2：長いタスクの中断

2000行以上のコードリファクタリングタスクを実行中。会議があって戻ってきたら—

Claude：どこやったか大致覚えてるが、細部を忘れた。
Codex：どこやったか大致覚えてるが、リファクタリングの方向がずれた。
Gemini：私は誰、どこにいる？

**それが当時の日常だった：ツールを切换するか、設定をやり直すか。**

---

## 02. RexCLIとは何か？

**RexCLI = Claude Code + Codex + Geminiが協調動作、互いに打架しない。**

### コア機能

| 機能 | 意味 |
|------|------|
| 統ブラウザ自動化 | どのCLIでも同じ`browser_*`コマンド |
| 跨CLIコンテキスト記憶 | ツールを切换しても進捗状況を保存 |
| Privacy Guard | 設定ファイルを自動脱敏、API Key泄露防止 |

### 動作原理

```
codex/claude/geminiを入力
       ↓
RexCLIが自動傍受
       ↓
判断：ラップするか通過させるか？
       ↓
ラップ：ContextDB + Browser MCPを接続
通過：元のツールに直接 전달
```

**作業習慣を変える必要はない。** 今までのように使えばいい。

---

## 03. 実際の効果

### 効果1：ブラウザ自動化

```bash
# 以前：Codexを使う
codex

# 今日：Codexを使っているが、能力が違う
codex
```

違いは、今なら統された`browser_*`ツールが使えること：
- `browser_navigate` - ページを開く
- `browser_click` - 要素をクリック
- `browser_snapshot` - ページコンテンツを取得
- `browser_screenshot` - スクリーンショット

**どのCLIに切换しても、これらのコマンドが使える。**

### 効果2：ブレークポイントからの再開

タスクの途中でツールを切换えたら？

```bash
# Codexで半分実行
codex

# Claudeに切换えて続行、コンテキスト自動同期
claude
```

**コピー＆ペースト要、タスクの再説明也要。**
