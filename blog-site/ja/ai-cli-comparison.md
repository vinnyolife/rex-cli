Claude Code、Codex CLI、Gemini CLI...

## 01. コンテキスト連続性（ブレークポイントからの再開）

これは**最も見落とされ、でも最も重要な**ディメンションだと思います。

### Claude Code：⭐⭐⭐⭐⭐

コンテキスト保持能力が**最も優秀**です。

2000 行のレガシーコードをリファクタリングした時、3日間放置してから再開しましたが、最後に関数をリファクタリングしていた場所をしっかり覚えていました。

`claude://` プロトコルが深く統合されており、ファイル変更履歴が清清楚楚です。

### Codex：⭐⭐⭐⭐

コンテキストもまあまあですが、ファイル間追跡能力は Claude よりわずかに劣ります。中規模タスクに適しています。

### Gemini：⭐⭐⭐

Google の強みは検索。プログラミングコンテキストは少し物足りません。長いタスクは「記憶喪失」を起こしやすいです。

**所感**：長いタスク（1 時間以上）をよく実行するなら、Claude Code を選びましょう。10 分以下の短いタスクしかしないなら、3つのツールの差はそんなにありません。

---

## 02. ブラウザ自動化能力

これはほとんどの比較記事が無視するディメンションです。

### Claude Code：⭐⭐⭐

MCP メカニズムはあるものの、組み込みブラウザ能力は一般的水準。自前でサードパーティ MCP を設定する必要があります。

**実測**：Playwright MCP の設定に30分かかり、実行中に不时に问题が発生しました。

### Codex：⭐⭐⭐⭐⭐

これは Codex の**ホームグラウンド**です。

`mcp-server/browser_*` ツールがネイティブサポート：スクリーンショット、クリック、フォーム入力、スクロール—オールインワンサービス。

そして検出回避做得也很好で、网站にバンされることはありません。

**実測**：小红書自動化を実行—がっちり稳定、3ヶ月使用中に一度も事故なし。

### Gemini：⭐⭐⭐

Google のブラウザ能力は主に Chrome DevTools Protocolで、技術デバッグ向けであり、日常自動化には向いていません。

**所感**：ブラウザ自動化が必要なら（データ収集、自動フォーム記入、バッチ操作）、**Codex を바로選びましょう**、悩む必要はありません。

## 03. ツール体験（対話流畅さ）

### Claude Code：⭐⭐⭐⭐⭐

最もスムーズ。CLI体験が最も自然な对话に近い。

Anthropic 製品一贯のエレガンス。ミシュラン星级レストランのような感じ—サービスが手的。

### Codex：⭐⭐⭐⭐

速くて、 명령어가 간결합니다。 でも対話体験が少し「机械的」。

ファストフードのような感じ—速く感じて吃饱できます。

### Gemini：⭐⭐⭐

Google の CLI は...怎么说呢。機能はまあまあだが、体験が少し粗糙い。

社員食堂のような感じ—期望 없이 가도 배는 채워줘요.

## 04. 障害回復（ブレークポイントからの再開）

### Claude Code：⭐⭐⭐⭐⭐

チェックポイントをサポート。回復メカニズムが最も完善。タスク途中で切れた？没关系、ブレークポイントから再開。

**実測**：リファクタリングタスクを実行中にノートパソコンがバッテリー切れ。電源をつないで再起動したら、最後の保存状態から続いて完了しました。

### Codex：⭐⭐⭐⭐

基本的な回復能力はあるが、Claudeほど詳細ではない。

### Gemini：⭐⭐⭐

回復メカニズムが比較的 기초적であり、長いタスク中断時は很容易出现问题。

## 05.  реальные сценарии сравнения

スペックだけを話してもつまらないので、直接シーンで説明します。

### シーン1：自动化小红书运营

| ツール | 体験 |
|-------|------|
| Claude Code | 自前でMCPを設定する必要があり、設定に30分、実行中不时に問題発生 |
| Codex | **箱から出してすぐ使えて、稳定运行、我每天在使用** |
| Gemini | このシーンには不太适合 |

**結論**：ブラウザ自動化は **Codex を選びましょう**。

### シーン2：複雑なコードリファクタリング

| ツール | 体験 |
|-------|------|
| Claude Code | コンテキスト理解が最も強く、リファクタリング思路清晰、コード品質が高い |
| Codex | 速いが、リファクタリング品質はClaudeにわずかに劣る |
| Gemini | 単純なタスクに適しているが、複雑なロジックは很容易跑偏 |

**結論**：複雑なリファクタリングは **Claude Code を選びましょう**。

### シーン3：素早いスクリプト作成

| ツール | 体験 |
|-------|------|
| Claude Code | あまりに优雅で、スクリプトには大炮で蝇を撃つようなもの |
| Codex | **丁度いい、命令简洁、速度快** |
| Gemini | 一応できるが、体験はまあまあ |

**結論**：素早いスクリプトは **Codex を選びましょう**。

### RexCLI が何を解决するか？

**問題1：ツール切り替え時にコンテキストが失われる**

以前：Codex で半分やっていて、Claude に最適화를 부탁하고 싶었으면—コンテキスト全部丢了，再次开始。

RexCLI の Context DB はタスク進捗状況を覚えていて、ツールを切换しても連続性が 끊れません。

**問題2：ブラウザ自動化設定が面倒**

각 도구마다 MCP 설정 방식이 달라서、面倒じゃないですか？

RexCLI は `browser_*` ツールを統一しました。Codex を使うか Claude を使うかに関わらず、同一コマンド一套でブラウザ操作が可能です。
