# 2026년, 어느 AI 프로그래밍 도구를 쓸지 고민 그만 — 실제로 잘 되는 방법을 알려드림

Claude Code, Codex CLI, Gemini CLI...

평가 영상을 잔뜩 봤는데, 점점 더 혼란스럽지 않나요?

걱정 마세요. 이 글은 벤치마크 수치나 과장 광고 따위 아닙니다. **어떤 상황에 어떤 도구를 꺼내야 하는지**, 그리고 **다 갖출 수 있는지**를 솔직하게 알려드릴게요.

## 01. 컨텍스트 연속성 (브레이크포인트에서 재개)

이건 **가장 무시되지만 가장 중요한** 차원이라고 생각해요.

### Claude Code: ⭐⭐⭐⭐⭐

컨텍스트 유지 능력이 **최강**이에요.

2000줄 레거시 코드를 리팩토링할 때 3일 동안 손도 못 대고 있다가 다시 시작했는데, 바로 마지막에 손 대던 함수부터 기억하고 있었어요.

`claude://` 프로토콜이 깊이 통합되어 있어서, 파일 수정 이력이해요.

### Codex: ⭐⭐⭐⭐

컨텍스트도 괜찮은데, 파일 간 추적 능력은 Claude보다해요. 중형 태스크에 적합.

### Gemini: ⭐⭐⭐

Google의 강점은 검색, 프로그래밍 컨텍스트는 좀 부족해요. 긴 태스크는 "기억 상실"되기 쉬워요.

**평가**: 긴 태스크(1시간 이상)를 자주 실행한다면 Claude Code를 선택하세요. 10분 이하 짧은 태스크만 한다면 세 도구 차이 별로 없어요.

---

## 02. 브라우저 자동화

이건 대부분의 비교 글이 다루지 않는 차원이에요.

### Claude Code: ⭐⭐⭐

MCP 메커니즘은 있긴 한데, 자체 브라우저 기능은 보통이에요. 타사 MCP 직접 구성해야 해요.

**실제 테스트**: Playwright MCP 설정하는 데 30분 걸렸고, 실행 중 가끔 문제 생겼어요.

### Codex: ⭐⭐⭐⭐⭐

이건 Codex의 **홈 그라운드**예요.

`mcp-server/browser_*` 도구가 네이티브로 지원돼요: 스크린샷, 클릭, 폼 채우기, 스크롤 — 올인원 서비스.

반면 检测做得도 잘 돼서, 웹사이트에서 밴 당하지 않아요.

**실제 테스트**: 소書 자동화 실행 — 확실하게稳稳, 3개월 사용 중 한 번도 문제 없었어요.

### Gemini: ⭐⭐⭐

Google의 브라우저 기능은 주로 Chrome DevTools Protocol, 기술 디버깅에는 적합한데 일상 자동화엔 부적합해요.

**평가**: 브라우저 자동화가 필요하면(데이터 수집, 자동 폼填写, 일괄 작업), **Codex 선택하세요**, 고민하지 마세요.

---

## 03. 도구 경험 (대화 유창성)

### Claude Code: ⭐⭐⭐⭐⭐

가장 유창해요. CLI 경험이 자연스러운 대화와 가장 가까워요.

Anthropic 제품 특유의 우아함. 미슐랭 스타 레스토랑 같은 느낌 — 서비스 철저.

### Codex: ⭐⭐⭐⭐

빠르고 명령이 간결해요. 그런데 대화 경험이 좀 "딱딱해요."

패스트푸드 같은 느낌 — 빠르고 든든해요.

### Gemini: ⭐⭐⭐

Google의 CLI는... 功能은 괜찮은데 경험이 좀 거칠어요.

직원 식당 같은 느낌 — 기대 없이 가도 배는 채워줘요.

---

## 04. 장애 복구 (브레이크포인트에서 재개)

### Claude Code: ⭐⭐⭐⭐⭐

체크포인트 지원, 복구 메커니즘이 가장 철저해요. 태스크 중간에 끊겼어요? 괜찮아요, 브레이크포인트부터 재개해요.

**실제 테스트**: 리팩토링 태스크 돌리는데 노트북 배터리 방전났어요. 전원 꽂고 재부팅하니까 마지막 저장 상태에서 이어서 완료했어요.

### Codex: ⭐⭐⭐⭐

기본적인 복구 능력은 있는데 Claude보다细节떨어져요.

### Gemini: ⭐⭐⭐

복구 메커니즘이 비교적 기초적이라, 긴 태스크 중단 시翻車.

## 05. 실제 시나리오 비교

스펙만 얘기하면 재미없으니 바로 시나리오로 갑시다.

### 시나리오 1: 소書 운영 자동화

| 도구 | 사용기 |
|------|--------|
| Claude Code | 직접 MCP 구성 필요, 설정에 30분, 실행 중 가끔 문제 |
| Codex | **바로 사용 가능, 안정적运行, 매일 쓰고 있어요** |
| Gemini | 이 시나리오엔太适合 |

**결론**: 브라우저 자동화는 **Codex 선택**.

### 시나리오 2: 복잡한 코드 리팩토링

| 도구 | 사용기 |
|------|--------|
| Claude Code | 컨텍스트 이해 최강, 리팩토링思路晰, 코드 품질 높음 |
| Codex | 빠르지만 리팩토링 품질은 Claude逊 |
| Gemini | 간단한 작업엔 적합, 복잡한 로직은 잘 튀어감 |

**결론**: 복잡한 리팩토링은 **Claude Code 선택**.

### 시나리오 3: 빠른 스크립트 작성

| 도구 | 사용기 |
|------|--------|
| Claude Code | 너무优雅해서, 스크립트엔杀鸡用牛刀 |
| Codex | **딱 좋��요, 명령 간결, 빨라요** |
| Gemini | 되긴 하는데 경험은 보통 |

**결론**: 빠른 스크립트는 **Codex 선택**.

## 06. 다 갖출 수 있을까?

있어요, 친구.

RexCLI 만들었어요, 오픈소스 워크플로 도구예요. 핵심思路: **새 바퀴 만들지 않고, 기존 Claude Code, Codex, Gemini를协同작업하게 함.**

### RexCLI가 뭘 해결하나요?

**문제 1: 도구 전환 시 컨텍스트 분실**

이전: Codex로 반쯤 하다가 Claude에게 최적화 부탁하고 싶어요?Unfortunately, 컨텍스트 다 날아가서 처음부터 다시 해야 해요.

RexCLI의 Context DB가 태스크 진행 상황을 기억해서, 도구 전환해도 연속성이 끊어지지 않아요.

**문제 2: 브라우저 자동화 설정이麻烦해요**

각 도구마다 MCP 설정 방식이 달라서, 귀찮지 않아요?

RexCLI가 `browser_*` 도구를 통일했어요. Codex를 쓰든 Claude를 쓰든,同套命令로 브라우저 조작 가능해요.

**문제 3:密钥 유출 위험**

스크립트 실행할 때 API Key가 실수로 로그에 새어나가지 않을까요?

RexCLI에는 Privacy Guard가 있어요. 설정 파일 읽기 전에 자동으로 탈敏해서, 키를 보호해줘요.
