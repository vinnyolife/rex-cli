# 장시간 작업 대결: Codex vs Claude vs Gemini — 실제로 승자는 누구인가?

세 가지 도구를 모두 사용해 본 후 내 결론: **최고의 도구는 없으며, 각 상황에 맞는 가장 적합한 선택지가 있을 뿐이다.**

하지만 문제가 있다. 대부분의 사람들은 어떤 상황에서 어떤 도구를 선택해야 하는지 알지 못한다.

오늘은 핵심만 다룬다. 솔직하게 이야기하겠다.

---

## 01. 빠른 결론

| 작업 유형 | 추천 도구 | 이유 |
|-----------|-----------|------|
| 복잡한 코드 리팩토링 | Claude Code | 가장 강력한 컨텍스트 이해력 |
| 브라우저 자동화 | Codex | 박스에서 꺼내서 바로 사용, 안정적 |
| 빠른 조사 | Gemini | 심층적인 Google 검색 연동 |
| **위 모든 것을 포함** | **RexCLI** | 크로스 도구 협업, 원활한 전환 |

---

## 02. 직접 비교

### 차원 1: 컨텍스트 연속성

**Claude Code**: 5점 만점에 5점

2000줄의 레거시 코드를 리팩토링하고 3일 동안 미루어 두었다가 돌아왔을 때, 정확히 어떤 함수에서 멈췄는지 기억하고 있었다.

**Codex**: 5점 만점에 4점

컨텍스트 유지 능력도 좋은 편이지만, 파일 간 추적 기능은 Claude보다 약간 떨어진다.

**Gemini**: 5점 만점에 3점

장시간 작업 중 "잊어버리는" 경향이 있다.

---

### 차원 2: 브라우저 자동화

**Claude Code**: 5점 만점에 3점

MCP를 수동으로 설정해야 한다. 30분이 걸렸다. 가끔 불안정하다.

**Codex**: 5점 만점에 5점

네이티브 `browser_*` 도구 지원: 스크린샷, 클릭, 폼 작성, 배치 처리. 3개월 동안 사용하면서 단 한 번도 문제가 없었다.

**Gemini**: 5점 만점에 3점

기술적 디버깅에는 좋지만 일상적인 자동화에는 부적합하다.

---

### 차원 3: 도구 사용 경험

**Claude Code**: 5점 만점에 5점

가장 매끄러운 상호작용. 미슐랭 스타 레스토랑 같다.

**Codex**: 5점 만점에 4점

빠르고 효율적이다. 패스트푸드처럼 필요할 때 바로 일을 끝낸다.

**Gemini**: 5점 만점에 3점

기능은 하지만 사용감이 거칠다.

---

### 차원 4: 장애 복구

**Claude Code**: 5점 만점에 5점

리팩토링 도중 노트북 배터리가 떨어졌다. 전원을 꽂고 재부팅했더니 정확히 멈춘 지점에서 이어서 작업했다.

**Codex**: 5점 만점에 4점

기본적인 복구 기능이 있지만, Claude보다는 세부 정보가 부족하다.

**Gemini**: 5점 만점에 3점

중단된 장시간 작업은 깔끔하게 재개되지 못하는 경우가 많다.

---

## 03. 당신은 무엇을 선택할 것인가?

### 상황: 소홍서 운영 자동화

- **Codex 선택**: 박스에서 꺼내서 바로 사용, 안정적, 매일 사용할 수 있다.

### 상황: 대규모 코드 리팩토링

- **Claude Code 선택**: 가장 강력한 컨텍스트 이해력.

### 상황: 빠른 스크립트 작성

- **Codex 선택**: 딱 맞고 빠르다.

### 상황: API 문서 조사

- **Gemini 선택**: 심층적인 Google 연동.
