# 장거리 작업에서 Codex, Claude, Gemini到底怎么选？

세 가지 도구를 써본 후 내 결론은: **가장 좋은 도구는 없고, 가장 맞는 상황이 있다.**

근데 문제—많은 사람들은 어떤 상황에 뭘 써야할지 모른다.

오늘은虚饰せず、直接教える。

## 02.维度 비교

###维度1: 컨텍스트 연속성

**Claude Code**: ⭐⭐⭐⭐⭐

제일誇张的是 これ 使った、代码改了、中间3天放置。再启动的时候居然记得上次重构到第几个函数了。

**Codex**: ⭐⭐⭐⭐

되지만跨파일 추적能力는稍弱。

**Gemini**: ⭐⭐⭐

장거리 작업容易"失忆"。

---

###维度2: 브라우저 자동화

**Claude Code**: ⭐⭐⭐

MCP 직접 설정 필요, 설정 30분,运行时偶爾抽风。

**Codex**: ⭐⭐⭐⭐⭐

`browser_*` 도구 네이티브 지원, 스크린샷, 클릭, 입력 일괄. 3달 运行没翻车。

**Gemini**: ⭐⭐⭐

기술 디버깅용, 일상 자동화엔 부적합.

---

###维度3: 도구 경험

**Claude Code**: ⭐⭐⭐⭐⭐

가장流畅, 미슐랭 레스토랑 같다.

**Codex**: ⭐⭐⭐⭐

빠른 서브웨이 같다.

**Gemini**: ⭐⭐⭐

직원 식당, 기대는 없지만 먹을만함.

---

###维度4: 장애 복구

**Claude Code**: ⭐⭐⭐⭐⭐

컴퓨터 전기 끊겼을 때, 껴서 재시작했더니 브레이크포인트에서 계속 해줬다.

**Codex**: ⭐⭐⭐⭐

기본적으로 되는데细则不如Claude.

**Gemini**: ⭐⭐⭐

장거리 작업 중단하면 잘 뻑남.
