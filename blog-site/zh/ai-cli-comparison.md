## 01. 上下文连续性（断点续跑能力）

这是我认为**最容易被忽视但最重要**的维度。

### Claude Code：⭐⭐⭐⭐⭐

上下文保持能力**最强**。

我用它改过一个 2000 行的遗留代码，中间隔了 3 天没动。再启动时，它居然记得我上次重构到第几个函数了。

`claude://` 协议深度集成，文件修改历史一清二楚。

### Codex：⭐⭐⭐⭐

上下文也不错，但跨文件追踪能力略弱于 Claude。适合中等长度的任务。

### Gemini：⭐⭐⭐

Google 家优势在搜索，编程上下文这块稍微差点。长任务容易"失忆"。

**我的评价**：如果你经常跑长任务（超过 1 小时），选 Claude Code。如果你只跑 10 分钟以内的任务，三者差别不大。

---

## 02. 浏览器自动化能力

这是很多测评不会告诉你的维度。

### Claude Code：⭐⭐⭐

有 MCP 机制，但自带浏览器能力一般。你需要自己配置第三方 MCP。

**实测**：配个 Playwright MCP 花了半小时，运行时偶尔抽风。

### Codex：⭐⭐⭐⭐⭐

这是 Codex 的**主场**。

`mcp-server/browser_*` 工具原生支持：截图、点击、填表、滚动，一条龙服务。

而且反检测做得不错，不会被网站 ban。

**实测**：跑小红书自动化，稳稳的，用了 3 个月没翻车。

### Gemini：⭐⭐⭐

Google 家浏览器能力主要在 Chrome DevTools Protocol，适合技术调试，不适合日常自动化。

**我的评价**：如果你要做浏览器自动化（抓数据、自动填表、批量操作），**直接选 Codex**，别折腾。

---

## 03. 工具体验（交互流畅度）

### Claude Code：⭐⭐⭐⭐⭐

交互最流畅，CLI 体验最接近自然对话。

Anthropic 家产品一贯的优雅。感觉像米其林餐厅，服务到位。

### Codex：⭐⭐⭐⭐

速度快，命令简洁。但交互体验稍微"硬"一点，像个理工男。

像快餐，速度快、管饱。

### Gemini：⭐⭐⭐

Google 家 CLI 一言难尽...功能性 OK，但体验有点糙。

像员工食堂，没啥期待，但能吃饱。

---

## 04. 故障恢复（断点续跑）

### Claude Code：⭐⭐⭐⭐⭐

支持 checkpoint，恢复机制最完善。任务中途断了？没关系，从断点继续。

**实测**：有次跑重构任务，电脑没电了。插电重启后，它居然从上次保存的状态继续跑完了。

### Codex：⭐⭐⭐⭐

有基本的恢复能力，但不如 Claude 细致。

### Gemini：⭐⭐⭐

恢复机制比较基础，长任务中断容易翻车。

---

## 05. 真实场景对比

光说参数没意思，直接上场景。

### 场景一：自动化小红书运营

| 工具 | 体验 |
|------|------|
| Claude Code | 需要自己配 MCP，配置半小时，运行时偶尔抽风 |
| Codex | **开箱即用，稳定运行，我每天在用** |
| Gemini | 不太适合这个场景 |

**结论**：跑浏览器自动化，**选 Codex**。

### 场景二：复杂代码重构

| 工具 | 体验 |
|------|------|
| Claude Code | 上下文理解最强，重构思路清晰，代码质量高 |
| Codex | 速度快，但重构质量略输 Claude |
| Gemini | 适合简单任务，复杂逻辑容易跑偏 |

**结论**：做复杂重构，**选 Claude Code**。

### 场景三：快速写个脚本

| 工具 | 体验 |
|------|------|
| Claude Code | 太优雅了，写脚本有点杀鸡用牛刀 |
| Codex | **刚刚好，命令简洁，速度快** |
| Gemini | 也行，但体验一般 |

**结论**：快速脚本，**选 Codex**。

## 06. 有没有可能全都要？

有的，兄弟。

我做了 RexCLI，一个开源工作流工具。核心思路是：**不重新造轮子，而是让现有的 Claude Code、Codex、Gemini 协同工作。**

### RexCLI 解决了什么问题？

**问题一：工具切换丢失上下文**

以前用 Codex 跑了一半，想让 Claude 帮忙优化一下？不好意思，上下文全丢了，得重来。

RexCLI 的 Context DB 会记住任务进度，切换工具不断档。

**问题二：浏览器自动化配置麻烦**

每个工具配置 MCP 的方式不一样，烦不烦？

RexCLI 统一了 `browser_*` 工具，不管你用 Codex 还是 Claude，都能用同一套命令操作浏览器。

**问题三：密钥泄露风险**

你跑脚本时，API Key 会不会不小心泄露到日志里？

RexCLI 有 Privacy Guard，配置文件读取前自动脱敏，保护你的密钥。

## 08. 彩蛋

其实对于大多数开发者，三个工具没那么大区别。

**真正影响效率的是工作流连续性**——你的任务能不能记住上次做到哪了，切换工具会不会丢上下文。

从这个角度，RexCLI 可能是目前最完整的方案。
