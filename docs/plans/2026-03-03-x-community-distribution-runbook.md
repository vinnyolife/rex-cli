# X / Community Distribution Runbook

## Objective

Turn each docs or blog update into measurable distribution output.

## Workflow

1. Produce content page.
2. Create channel-specific summary (X thread, GitHub Discussion, one forum post).
3. Attach one CTA and UTM tags.
4. Human review and publish.
5. Collect click and conversion metrics within 72 hours.

## Channel Policies

- X: concise hooks + threaded breakdown + one CTA.
- GitHub Discussions: technical depth + architecture context.
- External communities: follow each community promotion policy strictly.

## Role Split

- Agent: drafting, formatting, scheduling suggestions.
- Human: account login, final review, and publish approval.
