# English-First 15-Day Growth Sprint Design

## Goal

Increase site traffic and GitHub saves/stars with an English-first strategy, targeting 200 GitHub stars in 15 days.

## Product Positioning Guardrail

RexCLI is not a replacement for Codex/Claude/Gemini. It is an orchestration layer that adds:

1. Cross-CLI memory (ContextDB)
2. Browser automation (MCP)
3. Privacy-safe config reading (Privacy Guard)

All growth messaging must keep this framing.

## Audience

Primary audience:
- English-speaking developers already using CLI agents
- Builders running multi-tool workflows and automation

Secondary audience:
- Teams evaluating reliable AI coding workflows

## Distribution Design

Traffic and stars should come from a loop, not one-off posts:

1. Publish one clear English artifact (docs page, case, post)
2. Distribute in one or two channels (X, GitHub discussions/issues, relevant communities)
3. Route users to one conversion page with explicit star CTA
4. Record daily star/traffic numbers
5. Adjust next day based on signal

## Metrics Design

Primary KPI:
- GitHub stars: 0 -> 200 in 15 days

Supporting KPIs:
- Docs traffic (sessions/users)
- CTR from docs to GitHub repo
- Saves/bookmarks (proxy by stars and returning sessions)

Daily cadence:
- One update + one distribution action + one metric checkpoint.

## Execution Constraints

- English-first default for top-level pages and launch content.
- No broad channel spam; distribution must be technical and evidence-based.
- Keep claims concrete (demo, command, output, result).
