# Growth Information Architecture

## Objective

Design a documentation structure that captures intent-driven traffic and routes users to the right product destination.

## IA Layers

1. **Acquisition pages**: Home, Use Cases, pillar blog posts.
2. **Education pages**: Getting Started, architecture and workflow docs.
3. **Conversion pages**: destination CTAs to `cli.rexai.top`, `rexai.top`, `tool.rexai.top`.

## Navigation Model

- Top-level sections: Home, Getting Started, Use Cases, Community, Blog, SEO, Monetization.
- Each section ends with one explicit CTA.

## URL Strategy

- Keep slugs short and keyword-aligned.
- Avoid mixed-language slugs on top pages.
- Maintain stable canonical URLs to preserve ranking.
